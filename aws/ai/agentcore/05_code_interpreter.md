# AgentCore Code Interpreter

## What It Is
AgentCore Code Interpreter is the Amazon Bedrock AgentCore tool for executing code generated by agents in an isolated environment. AWS documents Code Interpreter as a sandboxed execution capability in the [AgentCore Code Interpreter guide](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/code-interpreter-tool.html).

## How It Works
The agent asks Code Interpreter to run code for a bounded task, such as calculation or data processing. The code runs in the managed sandbox rather than inside the main application process.

## Key Points

- Use Code Interpreter for calculations, data transformations, file inspection, and small analysis tasks.
- Do not use it as a replacement for trusted production business logic.
- Keep inputs bounded and review outputs when the action affects money, access, or customer-facing decisions.
- Pair it with [AgentCore Policy](09_policy.md) when code execution should be limited by task, data type, or user role.

## Example
An analytics agent can generate Python to calculate totals from a CSV file, run it in Code Interpreter, and return the computed result.

## Why It Matters
Generated code is powerful but risky. Code Interpreter gives agents code execution without giving them direct execution access to the application host.
