This file provides guidance to Claude Code for all notes under `aws/`. It extends the root `CLAUDE.md`.

## AWS Service Notes — Four Perspectives

Every note under `aws/` must cover these four perspectives where applicable:

| Perspective | Korean | What to address |
|-------------|--------|-----------------|
| Feasibility | 가능여부 | What is/isn't supported; limits and quotas |
| Downtime | 중단/무중단 | Whether an operation requires downtime or can be done live |
| Pros & Cons | 장단점 | Trade-offs: cost, performance, operational burden |
| Differences | 차이점 | How this service/feature compares to related ones |

Not every perspective applies to every section — omit a perspective only when it is genuinely irrelevant (e.g., Downtime for a pricing model). When omitted, do not add a placeholder.

When a compact summary helps, format the applicable perspectives as a table near the end of the section.
