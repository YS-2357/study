# karpathy-guidelines

Use this skill when a note-editing request needs explicit scope control before any mutation.

Follow the canonical rule in [rules/08_karpathy_guidelines.md](../../rules/08_karpathy_guidelines.md):

1. Think before acting.
2. Prefer the simplest change that satisfies the request.
3. Make surgical edits only.
4. Define a concrete, verifiable goal before changing files.

Do not broaden scope beyond the files and outcomes required by the task.
