# Plugins

## What It Is
A plugin is a larger bundle that can expose skills, tools, and integrations for an outside system.

Examples:
- GitHub
- Gmail
- Google Calendar
- Google Drive
- Vercel

## Analogy
If skills are playbooks, plugins are entire capability packs for a connected platform.

A GitHub plugin is not just one command. It may include:
- PR inspection tools
- issue tools
- review workflows
- CI-related skills

## Example
If you ask the agent to inspect a pull request, a GitHub plugin may let it:
- fetch PR metadata
- read review comments
- inspect changed files
- help draft or apply fixes

That goes beyond simple local file access.

## What People Mostly Use

For coding work, the most commonly useful plugins are:
- GitHub
- browser-related tooling
- cloud/deployment integrations
- document and spreadsheet integrations

People use plugins when they need the agent to work across systems, not just inside the repo.

## Plugin vs Skill

- Plugin = capability bundle for a platform
- Skill = reusable workflow guidance

A plugin may contribute several skills, but the two are not the same thing.

## Why It Matters

When someone says "the agent can work with GitHub," that usually means the harness has a GitHub-related plugin or tool integration available.
