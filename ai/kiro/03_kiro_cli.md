# Kiro CLI

## What It Is

Kiro CLI is the terminal version of Kiro. The deck presents it as a way to use Kiro's agentic workflow without leaving a shell-centered development environment.

## Analogy

If Kiro IDE is the full workshop, Kiro CLI is the same worker showing up at your command line instead of inside a graphical editor.

## How It Works

The deck's CLI slide keeps the message simple:

1. Run Kiro from the terminal.
2. Use it to implement ideas quickly and accurately.
3. Keep tight control over the process at each stage instead of handing everything to a black-box automation flow.

In practice, this makes CLI usage the shell-first counterpart to the IDE's spec-driven workflow.

## Example

A terminal-focused developer could stay inside the repo shell, invoke Kiro there, and still use Kiro as the active coding agent rather than switching into the full IDE UI for every task.

## Why It Matters

Many engineering workflows already live in the terminal. A CLI mode matters because it lets teams adopt Kiro's agent model without forcing the entire workflow into one editor surface.
