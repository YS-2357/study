## 섹션별 코멘트

### 컴퓨팅

- EC2 기본 설명은 적절함
- 인스턴스 패밀리, 속성, 명명 규칙은 101 기준으로 깊음
- 과금 옵션은 남겨도 되지만 설명 범위는 단순화하는 편이 좋음

### 스토리지

- S3 기본 개념은 적절함
- 스토리지 클래스 가격 비교는 가능하지만 “저장 비용 예시”임을 분명히 해야 함
- Intelligent-Tiering, S3 연계 서비스, EBS 세부 타입은 축약 권장

### 네트워크

- 네트워크 서비스 개요와 ELB 기본 설명은 적절함
- VPC 구성 요소와 ALB/NLB/GLB 비교는 101 기준으로 다소 상세함
- CloudFront도 수치보다 역할 중심으로 설명하는 편이 좋음

### 데이터베이스

- RDS 기본 개념은 적절함
- Aurora 소개는 짧게 가능하지만 Aurora vs RDS 상세 비교는 과함
- DynamoDB, ElastiCache는 “이런 서비스도 있다” 수준이면 충분함

### 보안

- 공동 책임 모델, IAM/WAF/Shield 개요는 101에 적절함
- IAM 세부 구조, WAF 세부 기능, Shield Advanced 상세 기능은 줄이는 편이 좋음
